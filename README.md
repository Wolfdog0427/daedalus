# Daedalus — Kernel Era

Daedalus is a governed, expressive runtime designed around one principle:  
**the operator is sovereign, the system is honest, and the body is coherent.**

The **Kernel era** is the sealed governance lineage of Daedalus. It defines:

- What Daedalus is allowed to see  
- What Daedalus is allowed to express  
- What Daedalus is allowed to remember  
- What Daedalus is never allowed to decide  

Once sealed, the Kernel era cannot be structurally changed. New features must live *around* it, not *inside* it.

---

## Kernel lineage (high level)

The Kernel era is composed of:

- **KernelShell** — boundary and intake gate  
- **KernelInvariants** — constitutional rules  
- **Kernel** — governance core  
- **KernelHalo** — introspection/snapshot surface  
- **KernelCrown** — expressive mapping of Kernel state  
- **KernelThrone** — cockpit view for the operator  
- **KernelRite** — activation ritual  
- **KernelSeal** — irreversible closure of the governance lineage

---

## What is sealed vs. extensible

**Sealed:**

- KernelShell, Kernel, KernelInvariants  
- Halo, Crown, Throne  
- All Integration Passes  
- Epistemic filters  
- Sovereignty logic  
- Node trust behavior  
- Continuity semantics  

**Extensible:**

- New apps  
- New panels  
- New tools  
- New expressive consumers
